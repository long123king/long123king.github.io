Immunity Debugger的组件很简单，它包含很少的几个PE文件，这些PE文件构成了GUI的骨架，里面的很多命令都是通过Python脚本扩展的。

daniel@daniel-mint ~/immdbg/Immunity Debugger $ ls -1 | xargs file | grep PE32
Bookmark.dll:         PE32 executable (DLL) (console) Intel 80386 (stripped to external PDB), for MS Windows
Cmdline.dll:          PE32 executable (DLL) (console) Intel 80386 (stripped to external PDB), for MS Windows
debugger.pyd:         PE32 executable (DLL) (console) Intel 80386 (stripped to external PDB), for MS Windows
ImmunityDebugger.exe: PE32 executable (GUI) Intel 80386 (stripped to external PDB), for MS Windows
loaddll.exe:          PE32 executable (GUI) Intel 80386, for MS Windows
uninstall.exe:        PE32 executable (GUI) Intel 80386 (stripped to external PDB), for MS Windows, Nullsoft Installer self-extracting archive

　　

其中核心的文件并不是ImmunityDebugger.exe，而是debugger.pyd。

 

这个debugger.pyd既是一个Windows DLL文件，可以被ImmunityDebugger.exe加载，同时也是一个Python的模块文件，可以在Python脚本中通过import导入。

其实ImmunityDebugger.exe/debugger.pyd都是先通过Python脚本编写的，然后通过py2exe转成二进制可执行文件的。
